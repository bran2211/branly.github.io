Analizador Lexico

Descripción

Este proyecto consiste en el desarrollo de un analizador léxico, una herramienta esencial en el proceso de compilación de lenguajes de programación. El analizador léxico, también conocido como "scanner," es el primer paso en la cadena de procesamiento de un compilador. Su función principal es leer el código fuente y dividirlo en una secuencia de tokens que representan las unidades léxicas del lenguaje, como palabras clave, identificadores, operadores, literales y símbolos.

Características Principales:
-Tokenización: El analizador léxico es capaz de identificar y categorizar diferentes tipos de tokens presentes en el código fuente, como palabras clave (if, while, int, etc.), operadores (+, -, *, /), y delimitadores ({, }, (, )).
-Gestión de Errores Léxicos: El sistema incluye una rutina de manejo de errores léxicos, capaz de detectar y reportar caracteres o secuencias no reconocidas en el código fuente, facilitando el proceso de depuración.
-Flexibilidad en la Definición de Tokens: El analizador permite la configuración de reglas léxicas mediante expresiones regulares, lo que facilita la adaptación del sistema a distintos lenguajes de programación o necesidades específicas.
-Integración con el Análisis Sintáctico: Este proyecto está diseñado para ser una pieza modular que se pueda integrar fácilmente con un analizador sintáctico, contribuyendo a un proceso de compilación completo.

  • Función 1: Analizar los caracteres.
  • Función 2: Dar resultados en tokens.
  • Función 3: Optimizar codigo.
  • Función 4: Guardar tokens.
